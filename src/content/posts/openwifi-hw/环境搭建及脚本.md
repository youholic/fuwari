## 前置条件准备

在开始编译之前，需要确保你的系统满足以下环境要求：

- **开发工具**：需要安装带有 Vitis 的 Vivado 2022.2 版本（注意：需要的是 Vitis，而不是 Vitis_HLS）。你可以通过 Vivado 菜单中的 "Add Design Tools for Devices 2022.2" 来添加。
    
- **许可证**：必须在 Vivado 中安装 Xilinx Viterbi Decoder 的评估版许可证。
    
- **操作系统**：推荐使用 Ubuntu 18/20/22 LTS 版本。
    
- **依赖库**：需要安装 `libtinfo5`，可以通过运行 `sudo apt install libtinfo5` 来完成（如果是 Ubuntu 24 LTS，则需要手动下载并安装 deb 包）。

## 逐步搭建步骤

**第 1 步：准备 Analog Devices HDL 库**（仅需运行一次）

- 设置 Xilinx 安装路径的环境变量：`export XILINX_DIR=your_Xilinx_install_directory`。
    
- 执行准备脚本：`./prepare_adi_lib.sh $XILINX_DIR`。

**第 2 步：准备特定开发板的 Analog Devices IP**（针对每种开发板仅需运行一次）

- 设置你的开发板名称变量：`export BOARD_NAME=your_board_name`（例如 `zc706_fmcs2`）。
    
- 执行准备脚本：`./prepare_adi_board_ip.sh $XILINX_DIR $BOARD_NAME`。

**第 3 步：获取 openofdm_rx IP**

- 将 openofdm_rx 获取到 ip 目录中：`./get_ip_openofdm_rx.sh`。

**第 4 步：为顶层 FPGA 项目生成 IP 库**

- 进入对应的开发板目录：`cd openwifi-hw/boards/$BOARD_NAME/`。
    
- 运行生成脚本：`../create_ip_repo.sh $XILINX_DIR`。此过程需要等待一段时间。

**第 5 步：在 Vivado 中生成比特流**

- 在 Vivado 控制台中运行：`source ../openwifi.tcl`。
    
- 在 Vivado GUI 界面点击 "Generate Bitstream"。
    
- 编译完成后，依次点击：File --> Export --> Export Hardware --> Next --> 勾选 Include bitstream --> Next --> Next --> Finish 进行硬件导出。

**第 6 步：存储 FPGA 文件**

- 返回 boards 目录：`cd openwifi-hw/boards`。
    
- 执行更新脚本：`./sdk_update.sh $BOARD_NAME $OPENWIFI_HW_IMG_DIR`。该命令会将 FPGA 的镜像文件（.xsa 和 .ltx）存储到特定的目录中，以便后续 openwifi 软件构建环境使用。

最后需要注意的是，虽然上述流程涵盖了 FPGA 硬件镜像的自动化构建，但 Openwifi ARM 端在构建时并没有用到自动化构建系统。

## 根目录下的脚本

### 1. `prepare_adi_lib.sh`
- 作用：准备 ADI（Analog Devices Inc.）HDL 库
- 功能：
  - 初始化并更新 `adi-hdl` 子模块
  - 切换到 `2021_r1` 分支
  - 编译 ADI 库（在 `library/` 目录执行 `make`）
  - 需要 Xilinx Vivado 环境

### 2. `prepare_adi_board_ip.sh`
- 作用：为指定开发板准备 ADI 板级 IP
- 功能：
  - 根据板名（如 `zcu102_fmcs2`、`zc706_fmcs2`、`adrv9361z7035` 等）选择对应的 ADI 项目目录
  - 编译对应板级的 ADI 项目

### 3. `get_git_rev.sh`
- 作用：获取当前 Git 仓库的最新提交哈希值
- 功能：
  - 输出短哈希（如 `abc1234`）
  - 若不在 Git 仓库中，返回 `ffffffff`

### 4. `get_ip_openofdm_rx.sh`
- 作用：获取 `openofdm_rx` IP 子模块
- 功能：
  - 初始化并更新 `ip/openofdm_rx` 子模块
  - 切换到 `dot11zynq` 分支并拉取最新代码

## `ip/` 目录下的脚本

### 5. `ip/create_vivado_proj.sh`
- 作用：创建 Vivado 项目
- 功能：
  - 调用 Vivado 执行 TCL 脚本创建项目
  - 支持最多 7 个参数传递给 TCL（板名、时钟频率、预定义宏等）

### 6. `ip/ultra_scale_tcl_gen.sh`
- 作用：为 UltraScale 系列 FPGA 生成 TCL 脚本
- 功能：
  - 查找包含 ZC706 相关配置的 TCL 文件
  - 复制并修改为 UltraScale 版本（将 `xc7z045ffg900-2` 替换为 `xczu9eg-ffvb1156-2-e`，将板卡信息替换为 ZCU102）

## `boards/` 目录下的脚本

### 7. `boards/pack_hdf_bit.sh`
- 作用：打包 HDF 和 bit 文件
- 功能：
  - 遍历所有支持的板卡
  - 将 `system.hdf` 和 `system_top.bit` 打包为 `hdf_and_bit.tar.gz`
  - 打包后删除原始文件以节省空间

### 8. `boards/create_ip_repo.sh`
- 作用：创建 IP 仓库
- 功能：
  - 为各 IP（xpu、tx_intf、rx_intf、openofdm_tx、openofdm_rx、side_ch）生成预定义文件（`*_pre_def.v`）
  - 支持为各 IP 设置条件编译宏定义
  - 调用 `ip_repo_gen.tcl` 生成 IP 仓库

### 9. `boards/sdk_update.sh`
- 作用：更新 SDK 目录
- 功能：
  - 将生成的 `system_top.xsa` 文件复制到目标 SDK 目录
  - 复制调试探针文件（`.ltx`）
  - 记录 Git 信息（分支、提交历史）到 `git_info.txt`

这些脚本用于自动化 OpenWiFi 硬件项目的构建流程，包括依赖准备、IP 生成、项目创建和 SDK 更新等步骤。

## 本项目中的 TCL 文件作用

TCL（Tool Command Language）是一种脚本语言。在 FPGA 开发中，Xilinx Vivado 使用 TCL 作为脚本语言，用于自动化项目创建、IP 配置、连接和构建流程。

### 1. IP 核创建脚本（`ip/*.tcl`）
- `xpu.tcl`、`rx_intf.tcl`、`tx_intf.tcl`、`openofdm_tx.tcl`、`side_ch.tcl` 等
- 作用：为各 IP 核创建独立的 Vivado 项目，配置源文件、约束和编译选项，生成可复用的 IP 核

### 2. 板级项目创建脚本（`boards/*/openwifi.tcl`）
- 作用：为特定开发板创建完整的 Vivado 项目
- 功能：
  - 创建项目并设置 FPGA 型号
  - 添加源文件和约束文件
  - 配置 IP 仓库路径
  - 设置综合和实现策略
  - 生成 Git 版本信息
  - 配置时钟频率等参数

### 3. IP 仓库生成脚本（`boards/*/ip_repo_gen.tcl`）
- 作用：为特定板卡生成 IP 仓库
- 功能：
  - 创建 `ip_repo` 目录
  - 生成配置宏定义文件（如 `clock_speed.v`、`board_def.v`、`spi_command.v` 等）
  - 调用 `package_ip.tcl` 打包各个 IP 核
  - 根据板卡类型设置 FPGA 规模标志

### 4. IP 连接脚本（`ip/connect_openwifi_ip.tcl`）
- 作用：将 OpenWiFi IP 核连接到系统设计中
- 功能：
  - 连接 AXI 接口到处理系统
  - 连接 ADC/DAC 数据路径
  - 连接中断信号
  - 连接时钟和复位信号

### 5. IP 定义脚本（`ip/openwifi_ip.tcl`）
- 作用：定义 OpenWiFi IP 的层次结构
- 功能：
  - 创建 IP 内部模块（xpu、rx_intf、tx_intf、openofdm_rx、openofdm_tx 等）
  - 配置 AXI 互连
  - 连接各模块间的信号

### 6. 板卡解析脚本（`ip/parse_board_name.tcl`）
- 作用：根据板卡名称解析 FPGA 型号和配置
- 功能：
  - 根据板卡名称（如 `antsdr`、`zcu102_fmcs2` 等）设置对应的 FPGA 型号
  - 设置是否为 UltraScale 系列标志
  - 设置 FPGA 规模标志（大/小）

### 7. IP 打包脚本（`boards/package_ip.tcl`、`boards/package_ip_openofdm_rx.tcl`）
- 作用：将 Verilog 源文件打包成 Vivado IP 核
- 功能：
  - 创建 IP 核项目
  - 添加源文件
  - 配置 IP 核属性
  - 生成 IP 核描述文件

### 8. 测试脚本（`ip/xpu/unit_test/*/*.tcl`）
- 作用：用于单元测试的仿真脚本

## 总结

这些 TCL 脚本构成了自动化构建流程：
1. 生成 IP 仓库（`ip_repo_gen.tcl`）
2. 创建板级项目（`openwifi.tcl`）
3. 连接 IP 核（`connect_openwifi_ip.tcl`）
4. 综合和实现生成比特流

通过 TCL 脚本，可以自动化完成从 IP 核创建到最终比特流生成的流程，减少手动操作并提高可重复性。

## 脚本执行顺序和依赖关系

### 阶段 1：环境准备（仅需执行一次）

```
1. prepare_adi_lib.sh
   └─ 作用：准备 Analog Devices HDL 库
   └─ 依赖：无
   └─ 输出：编译好的 adi-hdl/library

2. prepare_adi_board_ip.sh
   └─ 作用：为特定板卡准备 ADI IP
   └─ 依赖：prepare_adi_lib.sh（需要 ADI 库）
   └─ 输出：板级 ADI 项目

3. get_ip_openofdm_rx.sh
   └─ 作用：获取 openofdm_rx IP 子模块
   └─ 依赖：无
   └─ 输出：ip/openofdm_rx/
```

### 阶段 2：IP 仓库生成（每次构建前）

```
4. create_ip_repo.sh (boards/create_ip_repo.sh)
   └─ 调用：vivado -source ./ip_repo_gen.tcl
   │
   └─ 5. ip_repo_gen.tcl (boards/*/ip_repo_gen.tcl)
      ├─ 调用：exec ../../get_git_rev.sh
      ├─ 调用：source ../package_ip_openofdm_rx.tcl
      │   └─ 6a. package_ip_openofdm_rx.tcl
      │       └─ 调用：source ./openofdm_rx.tcl
      │           └─ 6b. openofdm_rx.tcl (ip/openofdm_rx/)
      │
      ├─ 调用：source ../package_ip.tcl (多次，针对不同 IP)
      │   └─ 7. package_ip.tcl
      │       └─ 用于打包：openofdm_tx, rx_intf, tx_intf, xpu, side_ch
      │
      └─ 内部调用各 IP 的 .tcl：
         ├─ 8. openofdm_tx.tcl (ip/openofdm_tx/)
         │   └─ 调用：source ../parse_board_name.tcl
         ├─ 9. rx_intf.tcl (ip/rx_intf/)
         │   └─ 调用：source ../parse_board_name.tcl
         ├─ 10. tx_intf.tcl (ip/tx_intf/)
         │    └─ 调用：source ../parse_board_name.tcl
         ├─ 11. xpu.tcl (ip/xpu/)
         │    ├─ 调用：source ../parse_board_name.tcl
         │    └─ 调用：exec ../../get_git_rev.sh
         └─ 12. side_ch.tcl (ip/side_ch/)
              └─ 调用：source ../parse_board_name.tcl
```

### 阶段 3：板级项目创建（在 Vivado 中执行）

```
13. openwifi.tcl (boards/*/openwifi.tcl)
    ├─ 调用：exec ../../get_git_rev.sh
    ├─ 依赖：ip_repo 目录（由 ip_repo_gen.tcl 生成）
    ├─ 依赖：adi-hdl 库（由 prepare_adi_lib.sh 生成）
    └─ 功能：
       ├─ 创建 Vivado 项目
       ├─ 添加源文件和约束
       ├─ 配置 IP 仓库路径
       └─ 打开 system.bd（Block Design）
           └─ 14. system.bd 内部会调用：
               ├─ openwifi_ip.tcl 或 openwifi_ip_ultra_scale.tcl
               │   └─ 创建 openwifi_ip 层次结构
               └─ connect_openwifi_ip.tcl 或 connect_openwifi_ip_ultra_scale.tcl
                   └─ 连接 IP 到系统
```

### 阶段 4：后处理（可选）

```
15. pack_hdf_bit.sh (boards/pack_hdf_bit.sh)
    └─ 作用：打包 HDF 和 bit 文件
    └─ 依赖：项目已生成 .hdf 和 .bit 文件

16. sdk_update.sh (boards/sdk_update.sh)
    └─ 作用：更新 SDK 目录，复制 .xsa 文件
    └─ 依赖：项目已生成 .xsa 文件
```

## 完整执行流程图

```
开始
 │
 ├─► [1] prepare_adi_lib.sh (一次性)
 │   └─► 编译 ADI 库
 │
 ├─► [2] prepare_adi_board_ip.sh (每个板卡一次)
 │   └─► 编译板级 ADI IP
 │
 ├─► [3] get_ip_openofdm_rx.sh (openofdm_rx 更新后)
 │   └─► 获取子模块
 │
 ├─► [4] create_ip_repo.sh
 │   └─► [5] ip_repo_gen.tcl
 │       ├─► get_git_rev.sh
 │       ├─► [6] package_ip_openofdm_rx.tcl
 │       │   └─► openofdm_rx.tcl
 │       └─► [7] package_ip.tcl (多次调用)
 │           ├─► [8] openofdm_tx.tcl
 │           ├─► [9] rx_intf.tcl
 │           ├─► [10] tx_intf.tcl
 │           ├─► [11] xpu.tcl
 │           └─► [12] side_ch.tcl
 │           └─► parse_board_name.tcl (被多个 IP 调用)
 │
 ├─► [13] openwifi.tcl (在 Vivado 中执行)
 │   ├─► get_git_rev.sh
 │   ├─► 创建项目
 │   └─► 打开 system.bd
 │       ├─► openwifi_ip.tcl
 │       └─► connect_openwifi_ip.tcl
 │
 ├─► [在 Vivado GUI 中]
 │   └─► 综合 → 实现 → 生成比特流
 │
 └─► [16] sdk_update.sh (可选)
     └─► 复制 .xsa 文件到 SDK 目录
```

## 关键依赖说明

1. `ip_repo_gen.tcl` 必须在 `openwifi.tcl` 之前执行，因为项目需要 `ip_repo` 目录。
2. `prepare_adi_lib.sh` 必须在 `openwifi.tcl` 之前执行，因为项目需要 ADI 库。
3. `get_git_rev.sh` 被多个脚本调用，用于生成版本信息。
4. `parse_board_name.tcl` 被多个 IP 的 `.tcl` 调用，用于解析板卡配置。

## 典型构建命令序列

```bash
# 1. 环境准备（仅一次）
export XILINX_DIR=/opt/Xilinx
./prepare_adi_lib.sh $XILINX_DIR
./prepare_adi_board_ip.sh $XILINX_DIR zc706_fmcs2
./get_ip_openofdm_rx.sh

# 2. 生成 IP 仓库（每次构建前）
cd boards/zc706_fmcs2/
../create_ip_repo.sh $XILINX_DIR

# 3. 在 Vivado 中创建项目
vivado
# 在 Vivado Tcl Console 中：
source ./openwifi.tcl
# 然后在 GUI 中点击 "Generate Bitstream"

# 4. 后处理（可选）
cd ../../
./boards/sdk_update.sh zc706_fmcs2 /path/to/sdk/dir
```
