---

## 三层架构详解

### 1. ARM 处理器层（顶部 - Linux 子系统）

运行 Linux 和 Wi-Fi 协议栈：
- Linux mac80211 subsystem：标准 Linux Wi-Fi 协议栈，提供 IEEE 802.11 接口
- SDR driver：软件定义无线电驱动集合，包括：
  - `rx_intf driver` / `tx_intf driver`：接收/发送接口驱动
  - `openofdm_rx/tx driver`：OFDM 接收/发送驱动
  - `xilinx_dma driver`：DMA 驱动，用于 ARM 与 FPGA 间高速数据传输
  - `xpu driver`：控制 FPGA 中的 XPU（扩展处理单元）
  - `ad9361 driver`：控制 AD9361 RF 收发器芯片

---

### 2. FPGA 层（中间 - 核心处理层）

包含三个主要模块：

#### 2.1 rx_intf（接收接口模块 - 左侧）

数据接收与处理：
- `adc_intf` / `rx_iq_intf`：从 RF 前端接收数字化的 I/Q 数据
- `openofdm_rx`：OFDM 解调与解码，输出数据包和原始 I/Q
- `rssi`：计算接收信号强度，传递给 `xpu`
- `pl_to_m_axis`：打包数据（含 RSSI、TSF 等），启动 DMA 传输到 Linux
- Xilinx AXIS DMA：与 ARM 层进行高速数据传输

#### 2.2 xpu（扩展处理单元 - 中间）

MAC 层控制与协议处理：
- `rx_parse`：解析接收到的数据包
- `pkt_filter`：数据包过滤
- `tx_control`：发送控制核心
  - 处理 ACK/CTS 回复
  - 管理重传
  - 等待 ACK
- `tsf_timer`：时间同步功能定时器
- `time_slice_gen`：时间片生成器（调度）
- `csma_ca`：载波侦听多路访问/冲突避免
  - 虚拟载波侦听
  - 退避算法
- `cca`：清空信道评估
  - 使用 RSSI 和前导码判断信道是否空闲
- `gpio_status` / `realtime gain`：与 RF 前端交互控制与状态

#### 2.3 tx_intf（发送接口模块 - 右侧）

数据发送处理：
- Xilinx AXIS DMA：从 ARM 层接收数据
- `tx_bit_intf`：等待 TXOP（传输机会），接收 `tx_control` 的控制信号
- `tx_queue 0` 到 `tx_queue N`：多个发送队列
  - 处理 CTS-to-Self 机制
  - 处理正常数据包发送
- `openofdm_tx`：OFDM 调制，生成 I/Q 数据
- `tx_iq_intf` / `dac_intf`：将 I/Q 数据发送到 RF 前端的 DAC

---

### 3. RF 前端层（底部 - 模拟信号处理）

Analog Devices AD9361 RF 收发器：

接收路径（左侧）：
- `rx1`, `rx2`：两个接收天线输入
- `adc`：模数转换器，将模拟 RF 信号转换为数字 I/Q
- `AGC`：自动增益控制，根据 `xpu` 的 `realtime gain` 调整接收增益

发送路径（右侧）：
- `dac`：数模转换器，将数字 I/Q 转换为模拟 RF 信号
- `tx1`, `tx2`：两个发送天线输出

---

## 数据流与控制流

### 接收数据流
```
RF天线 → ADC → FPGA (adc_intf → rx_iq_intf → openofdm_rx → rx_parse → pkt_filter) 
→ FPGA (pl_to_m_axis → m_axis → Xilinx AXIS DMA) 
→ ARM (rx_intf driver / xilinx_dma driver) 
→ Linux mac80211
```

### 发送数据流
```
Linux mac80211 → ARM (openofdm_tx driver / xilinx_dma driver) 
→ FPGA (Xilinx AXIS DMA → s_axis → tx_bit_intf → tx_queue → openofdm_tx) 
→ FPGA (tx_iq_intf → dac_intf) 
→ DAC → RF天线
```

### 控制流
- `xpu` 作为控制中心，通过 `tx_control`、`csma_ca`、`cca` 等模块实现 MAC 层协议
- `rssi` 信息从 `rx_intf` 传递到 `xpu` 用于 CCA（信道评估）
- `xpu` 通过 `gpio_status` 和 `realtime gain` 与 RF 前端交互

---

## 关键设计特点

1. 软硬件协同：ARM 处理高层协议，FPGA 处理实时 MAC 与物理层
2. 高速数据传输：通过 Xilinx AXIS DMA 实现 ARM 与 FPGA 间的高带宽传输
3. 实时性：MAC 层协议（CSMA/CA、ACK/CTS、重传等）在 FPGA 中硬件实现
4. 灵活性：基于 SDR，可软件配置与修改

该架构实现了完整的 IEEE 802.11 Wi-Fi 功能，同时保持了 SDR 的灵活性和可编程性。

```txt
RF天线 → ADC → FPGA (adc_intf → rx_iq_intf → openofdm_rx → rx_parse → pkt_filter) 
→ FPGA (pl_to_m_axis → m_axis → Xilinx AXIS DMA) 
→ ARM (rx_intf driver / xilinx_dma driver) 
→ Linux mac80211
```

```txt
Linux mac80211 → ARM (openofdm_tx driver / xilinx_dma driver) 
→ FPGA (Xilinx AXIS DMA → s_axis → tx_bit_intf → tx_queue → openofdm_tx) 
→ FPGA (tx_iq_intf → dac_intf) 
→ DAC → RF天线
```
