## 项目整体概述

**Openwifi** 是一个基于 SDR（软件无线电）的、完整开源的 **IEEE 802.11 a/g/n（Wi‑Fi 4）全栈实现**。  
它不是普通的“应用程序”，而是：自己实现了 Wi‑Fi 物理层/基带 + MAC 低层 + Linux 无线网卡驱动，使一块 FPGA+射频板（如 Zynq + AD9361）在 Linux 下表现成一块标准的 mac80211 兼容 Wi‑Fi 网卡 `sdr0`。

简单说：  
**这个仓库提供的是 openwifi 的软件侧实现（驱动 + 用户空间工具 + 启动/内核集成），硬件 FPGA 逻辑在另一个仓库 `openwifi-hw` 中。**

---

## 主要功能点

- **协议与性能**
  - 支持 **802.11a/g/n（Wi‑Fi 4）**，20MHz 带宽
  - 工作频段可配置为 **70MHz–6GHz**，支持子 GHz（2MHz for 802.11ah）、车联网 5.9GHz（10MHz for 802.11p）等场景
  - 支持 AP / Station / Ad‑hoc / Monitor 等模式
  - 实现 DCF（CSMA/CA）低 MAC 层在 FPGA 中，真正做到硬件级 10µs SIFS
  - 典型吞吐：TCP 40–50 Mbps，UDP 约 50 Mbps

- **特色能力**
  - **CSI（Channel State Information）导出**：可实时获取信道状态、频偏、等化器输出等，用于研究/雷达/感知
  - **CSI fuzzer / CSI radar**：可以伪造信道响应，做联合雷达与通信实验
  - **802.11 包注入和 fuzzing**：支持复杂的注入与测试场景
  - 可配置的 **信道接入参数**（CCA 阈值、SIFS/DIFS/slot time/CW 等），便于协议实验与研究
  - **时间切片（按 MAC 地址的时隙调度）**：可对不同终端做时间片资源调度

- **支持平台**
  - 支持多种基于 Xilinx Zynq/MPSoC + AD9361/AD9364 的开发板和第三方 SDR 板（如 ZC706+FMCOMMS2、ZedBoard、ADRV9364‑Z7020、ANTSDR、LibreSDR 等）

## 仓库内主要内容结构

- **`driver/`**  
  Linux 内核 **mac80211 软 MAC 驱动**，通过 `ieee80211_ops` 接口与 Linux 无线子系统对接，负责：
  - 收发帧中断处理（`openwifi_rx_interrupt` / `openwifi_tx_interrupt`）
  - 调用底层 FPGA 模块接口：`tx_intf` / `rx_intf` / `openofdm_tx` / `openofdm_rx` / `xpu` 等
  - 提供 testmode 接口给用户空间工具 `sdrctl` 使用

- **`user_space/`**  
  板端用户空间脚本和小工具，例如：
  - `wgd.sh`、`fosdem.sh`：一键启动/配置 AP、聚合等功能
  - `sdrctl_src/`：用户空间控制工具 `sdrctl` 源码，用于读写驱动/FPGA 寄存器、调节频率、功率、切片参数等
  - 大量辅助脚本：更新内核、更新驱动 ko、下发 FPGA bitstream、配置 hostapd/wpa_supplicant、抓 IQ、链路测试等

- **`kernel_boot/`**  
  与 Analog Devices Linux 内核及各开发板的 **设备树 / 启动镜像 / 补丁** 相关的文件，用于：
  - 生成适配各个 Zynq/MPSoC + AD936x 板卡的 BOOT.BIN、内核镜像、devicetree.dtb
  - 集成 openwifi 所需的 FPGA 外设映射、驱动加载等

- **`doc/`**  
  完整的项目文档，包括：
  - 项目整体架构图、模块说明（驱动 + FPGA + 用户空间）
  - 使用教程（烧录 SD 卡镜像、快速启动、基本操作）
  - 各种应用笔记：ad‑hoc、AP/client 示例、CSI、packet injection、雷达等
  - 端到端构建 Linux 镜像、porting 到新板卡的指南，以及发表论文与引用方式

- **`adi-linux/`、`adi-linux-64/`**  
  与 Analog Devices Linux 源码树配合，用于编译适配 openwifi 的内核和驱动（32/64 位架构）。

## 架构概览

架构分为三层：
1. **用户空间（User space）** - 顶层
2. **Linux内核空间（Linux Kernel space）** - 中间层
3. **软件定义无线电硬件（SDR Hardware）** - 底层

图例：白色/蓝色为 Linux 组件，黄色/蓝色为 Openwifi 组件。

## 各组成部分详解

### 1. 用户空间（User space）

- **数据应用（Data application）**：生成或消费数据流量的应用，通过系统调用与内核交互。
- **管理工具（Management）**：包含标准 Linux 工具（`wpa_supplicant`、`hostapd`、`iwconfig`、`iwlist`）和 Openwifi 的 `sdrctl`，用于配置和管理无线接口。

### 2. Linux 内核空间（Linux Kernel space）

#### 左侧栈（通用 Linux 网络栈）
- **system call**：用户空间与内核的接口
- **socket（协议无关）**：通用网络通信接口
- **network protocol（TCP/IP 等）**：标准网络协议实现
- **net_dev（设备无关）**：网络设备抽象层，提供统一接口
- **Ethernet driver**：以太网驱动

#### 右侧栈（802.11 专用栈）
- **nl80211**：Netlink 802.11 接口，供用户空间管理工具配置无线设备
- **cfg80211**：通用 802.11 配置 API，通过 `cfg80211_ops` 和“管理帧”与 `mac80211` 交互
- **mac80211**：IEEE 802.11 MAC 层实现，处理数据帧和管理帧
- **SDR driver（Openwifi）**：Openwifi 提供的驱动，通过 `ieee80211_ops` 与 `mac80211` 交互，连接 FPGA

### 3. SDR 硬件层

- **FPGA（Openwifi）**：可编程硬件，实现 Wi-Fi 信号处理
- **RF 前端（RF front-end）**：射频前端，处理模拟信号（放大、滤波、频率转换）
- **天线（Antenna）**：物理天线

## 数据流程

### 数据帧流程
```
数据应用 → system call → socket → network protocol → net_dev 
→ mac80211（数据帧）→ SDR driver → FPGA → RF 前端 → 天线
```

### 管理帧流程
```
管理工具（wpa_supplicant/sdrctl）→ nl80211 → cfg80211 
→ mac80211（管理帧 + cfg80211_ops）→ SDR driver → FPGA
```

## 总结

**这个项目的主要内容是：为多种 SDR+FPGA 硬件平台提供一个完整开源的 Wi‑Fi a/g/n 协议栈实现（含 Linux 驱动、用户空间工具、内核/启动集成和文档），让这些硬件在 Linux 下变成一块可高度可编程、可研究的 Wi‑Fi 网卡。**
