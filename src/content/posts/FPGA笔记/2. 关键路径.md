# 关键路径
## **1. 什么是关键路径？**
关键路径（Critical Path）是数字电路中**从输入到输出延迟最长的逻辑路径**，它决定了电路的**最高工作频率（F<sub>max</sub>）**。  
- **定义**：在时序分析中，关键路径是指组合逻辑中**传播延迟总和最大**的路径，即该路径的延迟直接限制了整个系统的时钟频率。
- **数学关系**：  
  $F_{max} = \frac{1}{T_{clk}} \quad \text{且} \quad T_{clk} \geq T_{su} + T_{comb\_max} - T_{skew}$
  - $T_{comb\_max}$ = 关键路径的组合逻辑延迟
如果关键路径延迟过长，系统可能无法在目标时钟频率下稳定工作。
我们在做时序优化（Timing Optimization）时，90% 的精力都是在“压缩”关键路径的延时。

---

## **2. 关键路径的识别**
在EDA工具（如Vivado、Quartus、Design Compiler）中，关键路径通常通过**静态时序分析（STA, Static Timing Analysis）** 报告显示。
工具会列出路径上的所有逻辑门、线延迟和时序裕量（Slack）。
例如：
```txt
Critical Path Delay: 12.3 ns (Max Frequency: 81.3 MHz)
Startpoint: reg_A (rising edge-triggered flip-flop)
Endpoint:   reg_B (rising edge-triggered flip-flop)
Path Group: clk
```

高扇出信号（如复位、时钟使能）会增加负载电容，导致延迟增大。  
**优化方法**：
- **插入缓冲器（Buffer）**：分担负载。

## **4. 关键路径优化流程**
1. **STA分析**：通过工具（如Vivado的 `report_timing`）定位关键路径。
2. **代码重构**：应用流水线、逻辑优化等方法。
3. **综合与布局布线**：选择优化策略（如 `-optimize_high`）。
4. **验证**：确保功能正确且时序收敛。
