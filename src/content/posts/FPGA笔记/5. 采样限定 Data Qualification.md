# Data Qualification

**Data Qualification** 就是**MUX/DMUX 同步法** 的正式学术名称或工程术语
## 1. 核心原理

**“Data Qualification”** 的核心思想是：**数据总线本身不经过同步器**，而是通过控制逻辑（Qualifier）来保证数据在被目的时钟采样时是**绝对静止**的。
- **Data Path（数据路径）：** 多比特数据直接从源寄存器连到目的寄存器（中间没有 FIFO，也没有打两拍）。
- **Control Path（控制路径/Qualifier）：** 一个单比特信号（通常叫 Valid, Enable, or Ready），经过“打两拍”同步到目的时钟域。
- **Qualification Logic（限定逻辑）：** 目的端只有在看到控制信号同步过来并有效后，才打开寄存器的使能端（Enable）去采样数据。
**为什么叫 "Qualification"？** 因为这个同步过来的控制信号起到了“资格认证”的作用——它告诉目的端：“现在数据已经稳定了，你有‘资格’去采它了”。

## 2. 电路结构 (The Architecture)

这种结构通常被称为 **"Recirculation MUX"（回环 MUX）** 结构。

1. **源域：** 发送数据 `Data_Src`，并保持数据**不变**。同时发送一个 `Tx_Valid` 信号。
2. **CDC 路径：**
    - `Data_Src` 走长线直接到目的端（不加触发器）。
    - `Tx_Valid` 经过两级触发器（2-FF Synchronizer）同步到目的时钟域，变成 `Rx_Valid`。
3. **目的域：**
    - `Rx_Valid` 经过边沿检测（Edge Detect）生成一个周期的脉冲 `Load_En`。
    - 目的寄存器带有一个 MUX（多路选择器）。
    - **如果不加载（Load_En=0）：** MUX 选择寄存器自己的输出（保持旧值，Recirculate）。
    - **如果要加载（Load_En=1）：** MUX 选择 `Data_Src` 进行更新。

## 3. 标准模板：
```txt
module data_qualification_cdc (
    input  wire clk_a,      // 源时钟
    input  wire clk_b,      // 目的时钟
    input  wire rst_n,
    input  wire [31:0] data_in,
    input  wire data_en_a,  // 源域数据有效信号（脉冲或翻转）
    output reg  [31:0] data_out
);

    // 1. 源域：寄存数据（为了防止毛刺）
    reg [31:0] data_src_reg;
    reg        valid_toggle; // 将脉冲转换为电平翻转，适合长距离传输
    
    always @(posedge clk_a or negedge rst_n) begin
        if (!rst_n) begin
            data_src_reg <= 32'd0;
            valid_toggle <= 1'b0;
        end else if (data_en_a) begin
            data_src_reg <= data_in;   // 数据更新
            valid_toggle <= ~valid_toggle; // 标志位翻转
        end
    end

    // 2. 控制路径 CDC：对 Toggle 信号打两拍
    reg valid_d1, valid_d2;
    always @(posedge clk_b or negedge rst_n) begin
        if (!rst_n) begin
            valid_d1 <= 1'b0;
            valid_d2 <= 1'b0;
        end else begin
            valid_d1 <= valid_toggle;
            valid_d2 <= valid_d1;
        end
    end

    // 3. 目的域：边沿检测（作为 Qualifier）
    reg valid_d3;
    wire load_enable;
    
    always @(posedge clk_b or negedge rst_n) begin
        if (!rst_n) valid_d3 <= 1'b0;
        else        valid_d3 <= valid_d2;
    end
    
    // 检测到 valid_d2 和 valid_d3 不同，说明源端发数据了
    assign load_enable = valid_d2 ^ valid_d3; 

    // 4. 数据路径：Data Qualification (MUX 结构)
    always @(posedge clk_b or negedge rst_n) begin
        if (!rst_n) 
            data_out <= 32'd0;
        else if (load_enable) // 只有当 Qualified 信号有效时才采样
            data_out <= data_src_reg;
        // else 保持不变 (隐含的 MUX Recirculation)
    end

endmodule
```
## 4. 关键约束 (强调)

为了让这种机制工作，**控制信号到达目的端并完成同步的时间，必须晚于数据到达的时间**。

也就是说，当 `load_enable` 变高去采数据的时候，数据必须早已经在 `data_src_reg` 输出端稳定了。
- **需要的约束：** `set_max_delay -datapath_only`
- **约束对象：** 针对 `data_src_reg` 到 `data_out` 的路径。
- **约束值：** 必须小于控制路径的最小延迟。通常设置为目的时钟周期的一半或更小，以确保安全。

## 5.**面试：**

"Data Qualification 是一种节省资源的多比特 CDC 方案，通过同步一个控制信号（Qualifier）来指示目的端何时可以安全地采样保持稳定的数据总线，常配合 `set_max_delay` 约束使用。"
