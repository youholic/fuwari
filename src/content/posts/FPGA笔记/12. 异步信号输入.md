# 一、 常见的异步信号场景 (Where?)

在FPGA设计中，异步信号主要来源可以分为以下三类：
## 1. 物理接口输入 (Physical IO)
这是最直观的异步信号，因为外部世界不跟随FPGA的时钟跳动。

- **按键/开关 (Buttons/Switches)：** 极慢速，且伴随机械抖动。
- **低速通信接口 (UART, I2C, SPI)：** 虽然它们有波特率或时钟，但相对于FPGA的主时钟（如100MHz），它们的数据到达时间是异步的。
- **中断信号 (Interrupts)：** 外部传感器或MCU发出的中断请求。
- **复位信号 (Reset)：** 外部按键复位通常是异步的 **[13. 复位信号处理](/posts/src\content\posts\FPGA笔记\13. 复位信号处理/)**。
- **为什么异步复位信号很特殊？**
	如果复位信号直接连到所有寄存器的异步复位端（CLR），会有一个致命隐患：**恢复时间（Recovery Time）违例**。
	- **场景：** 当复位信号**撤销（变无效，释放复位）** 的那一瞬间，如果恰好发生在时钟上升沿的附近。
	- **后果：** 触发器内部会发生亚稳态。导致的结果是：FPGA中**一部分寄存器成功解除了复位开始工作，而另一部分寄存器还处于复位状态**。
	- **现象：** 状态机跑飞、系统随机挂死。
## 2. 跨时钟域交互 (Internal CDC)

随着FPGA设计变大，内部往往存在多个时钟域。

- **时钟频率不同：** 例如，采集模块跑在 50MHz，处理模块跑在 200MHz。
- **时钟同频但不同相：** 两个时钟都是 100MHz，但源自不同的PLL输出或晶振，相位关系不固定。
- **数据流处理：** 视频流（Pixel Clock）进入内存接口（DDR Clock）。
## 3. 不同的物理芯片互联 (Chip-to-Chip)

- **ADC/DAC 接口：** 高速ADC通常输出随路时钟（Source Synchronous），这个时钟和FPGA内部逻辑时钟是异步的。
- **异步存储介质访问**（Asynchronous Memory Access）

# 二、异步信号分类

如果我们要给“数据传输产生的异步”做一个最严谨的定义，可以总结为以下四种模式：

#### ① 频率/相位异步 (Clock Domain Crossing)

- **本质：** 两边都有时钟，但时钟不同源。
- **例子：** FPGA 内部 `Clk_A` 到 `Clk_B`。
- **关键：** 解决亚稳态。

#### ② 采样异步 (Asynchronous Sampling)

- **本质：** 发送方没有时钟，接收方用自己的时钟去“硬采”。
- **例子：** 串口 UART、按键、外部中断信号。
- **关键：** 解决信号抖动、毛刺以及采样位置的选择。

#### ③ 源同步异步 (Source Synchronous)

- **本质：** 发送方带着时钟来，但这个时钟和接收方的系统时钟是“亲戚”甚至完全没关系。
- **例子：** DDR 内存（DQS 信号）、高速 ADC (DCLK)。
- **关键：** 解决**相位对齐（Phase Alignment）**。这不只是 0 和 1 的问题，而是要让采样点精准落在数据的中心。

#### ④ 协议/时序异步 (Protocol/Timing Asynchronous)

- **本质：** 信号的变化遵循一套时间参数（如 $t_{AS}$ 建立时间，$t_{RC}$ 周期时间），而不是时钟边沿。
- **例子：** 访问并口 SRAM、驱动外部并行总线。
- **关键：** 建立**静态时序模型**，手动分配逻辑周期。

为什么理解这些“需求”很重要？因为**处理手段的开销完全不同**：

1. 如果是 **模式①**（内部CDC），你可以用 **异步FIFO**，虽然费资源但最稳。
2. 如果是 **模式②**（接口输入），你只能用 **2-FF + 边缘检测**，因为你根本没有对方的时钟可以参考。
3. 如果是 **模式③**（高速源同步），你必须动用 FPGA 的**硬核资源（IDELAY/PLL）**，因为逻辑层面的 2-FF 已经太慢了，无法保证时序收敛。

# 三、 物理接口输入处理

处理外部非复位信号（如 UART RX, I2C SCL/SDA, SPI MISO/MOSI, GPIO中断等），通常遵循一套标准的 **输入处理流水线**。
这套流水线通常包含三个步骤：**同步（Sync） -> 滤波（Filter/Debounce） -> 边沿检测（Edge Detect）**。
无论是什么低速接口信号，进入FPGA后的头几行代码通常都是长这样的：
## 1. 必须同步 (Mandatory Synchronization)

不管外部信号看起来多稳定，相对于FPGA内部时钟，它都是异步的。

- **操作：** 所有进入FPGA逻辑的IO信号，**第一件事**就是通过两级触发器（2-FF）。
- **目的：** 消除亚稳态。
- **代码：**
```
always @(posedge sys_clk) begin
	sync_d1 <= async_input_pin;
	sync_d2 <= sync_d1; // sync_d2 才是我们在内部使用的信号
end
```

## 2. 根据类型进行“净化” (Purification)


同步后的信号可能还带有毛刺，或者我们需要判断逻辑上的“有效性”。这里分两种情况：

1. **机械/开关类信号（按键、拨码）：**

    - **处理方式：** **Debounce（消抖）**。
    - **原因：** 机械触点会有几十毫秒的物理抖动。
    - **逻辑：** 见之前的解释，使用计数器，只有信号稳定保持 N ms 才认可。

2. **通信协议类信号（UART, I2C, SPI）：**

    - **处理方式：** **数字滤波 (Digital Filter) / 多数表决**（可选但推荐）。
    - **原因：** 信号线上的电磁干扰可能产生极窄的尖峰（Glitch）。
    - **逻辑：** 连续采样3次，比如 `111` 才是1，`000` 才是0，中间状态保持原值。

 **Debounce（消抖）** 和 **Qualifier（滤波/确认）** 

都是针对输入信号进行处理的技术，目的都是为了**去除毛刺和干扰，确保采集到稳定的信号**。
### a. Debounce（消抖）

**关键词：** 机械开关、毫秒级 (ms)、大计数器

- **1. 产生原因**
主要用于处理机械按键或开关。
当按下或释放一个机械按键时，金属触点接触的瞬间会发生弹性碰撞，导致电平在短时间内（通常为 5ms ~ 20ms）剧烈抖动。如果直接采样，FPGA 会误认为按下了几十次键。
- **2. 处理方法**
	**核心逻辑：** “只有当信号在保持了一个较长的时间（如 20ms）不变后，我才认为它是有效的。”
	通常使用一个**计数器**来实现：
	1. 检测到输入变化。
	2. 启动计数器开始计时。
	3. 如果在计时过程中输入又变了，清零计数器重新计时。
	4. 只有计数器计满了（例如计满 20ms），才更新输出状态。
- **3. Verilog 代码示例 (简易版)**
```c
module debounce (
    input wire clk,
    input wire rst_n,
    input wire key_in,      // 原始机械信号
    output reg key_out      // 消抖后的信号
);

    parameter CNT_MAX = 1_000_000; // 假设50MHz时钟，20ms = 1,000,000个周期
    reg [19:0] cnt;
    reg key_in_sync_0, key_in_sync_1;

    // 先做打两拍同步 (CDC处理)，防止亚稳态
    always @(posedge clk or negedge rst_n) begin
        if (!rst_n) begin
            key_in_sync_0 <= 1'b1;
            key_in_sync_1 <= 1'b1;
        end else begin
            key_in_sync_0 <= key_in;
            key_in_sync_1 <= key_in_sync_0;
        end
    end

    // 消抖逻辑
    always @(posedge clk or negedge rst_n) begin
        if (!rst_n) begin
            cnt <= 0;
            key_out <= 1'b1;
        end else begin
            if (key_in_sync_1 != key_out) begin
                // 如果输入和当前输出不同，说明在变化，开始计数
                if (cnt < CNT_MAX) 
                    cnt <= cnt + 1;
                else begin
                    // 计数满了，说明稳定了，更新输出
                    cnt <= 0;
                    key_out <= key_in_sync_1;
                end
            end else begin
                // 输入如果变回去了（抖动），计数器清零
                cnt <= 0;
            end
        end
    end
endmodule
```
### b. Qualifier（信号确认/滤波）

**关键词：** 电气噪声、纳秒/微秒级 (ns/us)、移位寄存器/小计数器
-  **1. 产生原因**
主要用于处理电气干扰或毛刺 (Glitch)。
在数字信号传输中（如 I2C, SPI, 串口或传感器输入），可能会因为线路串扰或辐射产生极短的尖峰脉冲（比如 1-2 个时钟周期的毛刺）。这些不是机械抖动，但也会导致逻辑错误。
- **2. 处理方法**
	**核心逻辑：** “连续 N 个时钟周期都检测到高电平，我才认为是高电平。”
	Qualifier 通常也被称为 **Input Glitch Filter**。它不需要像 Debounce 那样等几十毫秒，通常只需要几个时钟周期。
	**常见实现：**
	- **移位寄存器法：** 连续采样 N 次（比如 4 次），只有当这 4 位全是 1，输出才变 1；全是 0，输出才变 0。
- **3. Verilog 代码示例 (4-bit Qualifier)**
```c
module qualifier (
    input wire clk,
    input wire rst_n,
    input wire signal_in,
    output reg signal_out
);

    reg [3:0] shift_reg;

    always @(posedge clk or negedge rst_n) begin
        if (!rst_n) begin
            shift_reg <= 4'b0000;
            signal_out <= 1'b0;
        end else begin
            // 移位寄存器，左移并在最低位补入新值
            shift_reg <= {shift_reg[2:0], signal_in};

            // 只有连续4次都是1，才输出1
            if (shift_reg == 4'b1111)
                signal_out <= 1'b1;
            // 只有连续4次都是0，才输出0
            else if (shift_reg == 4'b0000)
                signal_out <= 1'b0;
            // 否则保持 signal_out 不变（过滤掉中间态毛刺）
        end
    end
endmodule
```

## 3. 边沿检测 (Edge Detection) —— **最关键的一步**

这是新手最容易犯错的地方。 **千万不要**直接把外部接口的时钟（如 SPI_CLK, I2C_SCL）当作FPGA内部逻辑的 `always @(posedge ...)` 时钟来用！

- **错误做法：** `always @(posedge i2c_scl_pin) ...` （这会产生多时钟域噩梦，且抗干扰极差）。
- **正确做法：** 使用FPGA的主时钟（如50MHz）去**采样**外部信号，通过前后两拍的状态对比来**“计算”**出上升沿或下降沿。
- **代码定式：** 此后，你的所有逻辑（状态机、计数器）都基于 `sys_clk` 和 `is_rising_edge` 标志位来跑。
``` c
reg [1:0] edge_detect;
always @(posedge sys_clk) begin
    // 假设 signal_synced 是已经同步好的信号
    edge_detect <= {edge_detect[0], signal_synced}; 
end

// 组合逻辑判断边沿
wire is_rising_edge  = (edge_detect == 2'b01); // 上一拍0，这一拍1
wire is_falling_edge = (edge_detect == 2'b10); // 上一拍1，这一拍0
```

## 4. 这种方法有什么限制/特例

这种“过采样（Oversampling）”方法要求：**FPGA的主时钟频率必须至少是外部信号频率的 3~4 倍以上**（根据香农采样定理，最好是 10 倍以上以保证相位精度）。

- 例如：外部SPI时钟是 10MHz，FPGA主时钟是 100MHz -> **完全没问题，用上述方法。**
- 例如：外部SPI时钟是 100MHz，FPGA主时钟是 100MHz -> **上述方法失效。** 需要使用专门的硬件原语（如 IDDR 寄存器）或作为源同步时钟处理。

### 特例 1：外部时钟输入 (External Clock Input)

**场景：** 晶振、外部芯片提供的参考时钟。

**为什么特殊？**

1. **不能当数据处理：** 你绝对不能把时钟输入引脚当作普通信号去打两拍（2-FF）。
2. **必须走专用网络：** 时钟信号必须驱动FPGA内部成千上万个触发器。如果走普通布线（General Routing），会导致极大的**时钟偏斜（Clock Skew）**，导致时序违例。

**怎么处理？**

- **硬件原语：** 必须通过 **IBUFG (Global Clock Input Buffer)** 或类似的缓冲器进入。
- **后续连接：** 必须连接到 **BUFG (Global Clock Buffer)** 或 **PLL/MMCM**。
- **作用：** 这些原语会将信号推上FPGA的“全局时钟树（Global Clock Tree）”，确保到达芯片每一个角落的延迟几乎一致。

**代码示例（通常由IP核或原语调用）：**

```c
// 错误：直接用 clk_pin 驱动逻辑
// always @(posedge clk_pin) ... 

// 正确：先通过专用Buffer
wire sys_clk_internal;
IBUFG u_ibufg (
    .I (clk_pin),
    .O (sys_clk_internal) // 这个信号才能去驱动 always @(posedge ...)
);
```

### 特例 2：双向信号 (Bi-directional / Inout)

**场景：** I2C 数据线 (SDA)、SPI (3-wire 模式)、DDR 数据线、SD卡数据线。

**为什么特殊？**

- **物理冲突风险：** 这种引脚既能输入也能输出。如果处理不好（例如外部驱动低电平，FPGA强行驱动高电平），会导致**短路**，甚至烧毁引脚。
- **Verilog 语法：** 不能简单定义为 `input` 或 `output`，必须定义为 `inout`。

**怎么处理？**

- **硬件结构：** 必须使用 **三态缓冲器 (Tri-state Buffer)** 结构，通常对应原语 `IOBUF`。
- **控制逻辑：** 需要一个“输出使能信号”（Output Enable, OE/Tri）。
    - 当 `OE = 1`：FPGA 驱动引脚（输出）。
    - 当 `OE = 0`：FPGA 输出呈高阻态（Z），此时引脚状态由外部决定，FPGA 仅作为输入读取。


**代码定式（三态门）：**

```c
module i2c_io (
    inout  wire sda_pin,  // 物理管脚
    input  wire tx_data,  // 内部想发的数据
    input  wire tx_en,    // 发送使能 (1=发, 0=收)
    output wire rx_data   // 内部收到的数据
);

    // 关键语法：三态赋值
    // 如果使能有效，输出 tx_data；否则输出高阻态 'z'
    assign sda_pin = (tx_en) ? tx_data : 1'bz;

    // 任何时候都可以读取引脚的真实电平
    assign rx_data = sda_pin;

endmodule
```

### 特例 3：高速源同步信号 (High-Speed Source Synchronous)

**场景：** 高速 ADC/DAC 接口、DDR 内存接口、摄像头接口（CSI/DVP）、RGMII 千兆网口。

**为什么特殊？**

- **太快了：** 信号变化速度极快（例如 500Mbps），数据有效窗口（Data Valid Window）极小。
- **相位对齐：** 外部芯片发送数据的同时，会送过来一个随路时钟（Clock）。**数据是跟这个随路时钟对齐的，而不是跟 FPGA 的主时钟对齐的。**
- **普通IO搞不定：** 如果你用普通的 2-FF 同步，连第一级触发器的建立/保持时间（Setup/Hold Time）都满足不了，数据直接采错。

**怎么处理？**

1. **使用 IDDR / ISERDES：** 使用 FPGA IO Block 里的专用硬件资源（Input Double Data Rate 寄存器）。它们位于 I/O Pad 旁边，路径延迟最小。
2. **动态相位调整 (Delay Line / IDELAY)：** 使用 `IDELAY` 原语，微调数据或时钟的延迟（以皮秒 ps 为单位），让时钟沿精确地对准数据的中心。
3. **处理流程：**
    - 时钟进入 PLL/MMCM 生成采样时钟。
    - 数据进入 IDDR 采样（此时数据还没跨时钟域）。
    - 采样后的数据通常是并行化的（降速了），然后再通过 FIFO 跨到 FPGA 的系统时钟域。

### 特例 4：差分信号 (Differential Inputs)

**场景：** LVDS (Low Voltage Differential Signaling)、HDMI、PCIe、SATA。

**为什么特殊？**

- **物理形态：** 外部是两根线（P 和 N），但在 FPGA 逻辑内部，我们只处理一个单比特信号（0 或 1）。
- **电气特性：** 电压摆幅很小（几百毫伏），普通 CMOS 接口无法识别。

**怎么处理？**

- **IBUFDS 原语：** 必须在顶层模块实例化差分输入缓冲器。它把物理上的 `_p` 和 `_n` 两根线，“合并”成逻辑上的一个信号。

**代码定式：**
```c
// 必须在 Top 层例化，不能用逻辑运算 assign a = p - n;
IBUFDS #(
    .DIFF_TERM("TRUE"),    // 打开内部匹配电阻
    .IOSTANDARD("LVDS_25") // 指定电平标准
) u_ibufds (
    .I  (data_p_pin), // 物理正端
    .IB (data_n_pin), // 物理负端
    .O  (data_internal) // 转换后的单端逻辑信号 -> 之后再按标准流程处理
);
```

# 四、总结举例

| **信号类型**  | **关键特征** | **处理核心**               | **典型原语/代码**                               |
| --------- | -------- | ---------------------- | ----------------------------------------- |
| **复位信号**  | 异步且敏感    | **Reset Bridge**       | `always @(posedge clk or negedge rst_in)` |
| **普通慢信号** | 异步且有毛刺   | **2-FF Sync + Filter** | `reg [1:0] sync;`                         |
| **时钟信号**  | 驱动大量负载   | **全局时钟网**              | `IBUFG` -> `BUFG` / `PLL`                 |
| **双向信号**  | 既入又出     | **三态门控制**              | `assign pin = en ? data : 1'bz;`          |
| **高速源同步** | 随路时钟，极快  | **IDDR + 延时调整**        | `IDDR`, `IDELAY`, `FIFO`                  |
| **差分信号**  | 双线表示1bit | **差分转单端**              | `IBUFDS`                                  |
#### 场景 1：处理 UART RX 信号

UART是典型的异步通信。

1. **Sync:** `rx_pin` -> `rx_sync` (2-FF)。
2. **Edge Detect:** 在空闲状态检测 `rx_sync` 的**下降沿**（Start Bit）。
3. **Sampling (过采样):** 一旦检测到下降沿，启动一个计数器。
    - 假设波特率9600，FPGA时钟50M。算出每个bit的周期数。
        
    - 在每个bit周期的**中间位置**（Middle point）去采样数据，这样最稳定。
        
    - _通常我们会使用 16倍波特率 的频率去采样，然后在中间几拍做多数表决。_
#### 场景 2：处理 SPI / I2C 从机接口 (Slave Mode)

假设你要做一个SPI Slave，外部主机发来 `SCLK` 和 `MOSI`。

1. **Sync:** `SCLK` -> `sclk_sync`; `MOSI` -> `mosi_sync`。
2. **Edge Detect:** 检测 `sclk_sync` 的上升沿标志 `sclk_rise`。
3. **Logic:**
```
always @(posedge sys_clk) begin
    if (cs_n_sync == 0) begin // 片选有效
        if (sclk_rise) begin  // 只有在检测到外部时钟上升沿的那个周期
            shift_reg <= {shift_reg[6:0], mosi_sync}; // 移位采样
        end
    end
end
```

**注意：** 整个逻辑依然完全运行在 `sys_clk` 域下，外部 `SCLK` 只是作为一个数据信号被采样，用来生成使能脉冲。
#### 场景 3：外部中断 (Interrupt Input)

外部传感器给一个脉冲告诉FPGA“有数据了”。

1. **Sync:** `intr_pin` -> `intr_sync`。
2. **Edge Detect:** 检测 `intr_sync` 的上升沿得到 `intr_pulse`。
3. **Logic:** 将 `intr_pulse` 连接到内部状态机的触发条件上。
