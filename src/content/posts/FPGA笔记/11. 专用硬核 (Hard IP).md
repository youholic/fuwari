在 FPGA 中，**专用硬核 IP (Dedicated Hard IP)** 是指那些**在芯片出厂时就已经固化在硅片上的、功能固定的 ASIC 电路模块**。
# 一、 核心计算与存储类 (基础三件套)

这些资源在芯片内部呈**列状 (Column)** 分布，几乎每个 FPGA 设计都会用到。

## 1. 块存储器 (Block RAM / BRAM)
- **是什么：** 嵌入在 FPGA 内部的高速 SRAM 块。在 Xilinx 7 系列中，一个 BRAM 大小通常是 **36Kb**（可拆分为两个 18Kb）。
- **用途：** 实现 FIFO（跨时钟域）、ROM（存系数表）、RAM（存数据缓存）。
- **硬核优势：** 如果用 LUT 搭建同样的 36Kb 存储器，会消耗成千上万个 CLB，而且布线会乱成一团麻，速度极慢。BRAM 速度快且时序确定。
- **进阶：** 在 UltraScale+ 系列中，还有 **UltraRAM (URAM)**，容量更大（288Kb），专门用于替代外部存储器做大缓存。
## 2. 数字信号处理单元 (DSP Slice)

- **是什么：** 专用的算术运算单元。常见型号如 **DSP48E1 / DSP48E2**。
- **内部结构：** 包含 **25x18 乘法器**、**48位累加器**、预加器 (Pre-adder) 和模式检测逻辑。
- **用途：** 所有的数字信号处理算法（FIR 滤波器、FFT、矩阵运算）。
- **硬核优势：** 一个乘法器如果用 LUT 搭，极其消耗资源且不仅跑不快。DSP Slice 可以轻松跑在 500MHz+，是 FPGA 做信号处理的核心资本。
## 3. 时钟管理单元 (CMT / MMCM / PLL)

- **是什么：** 模拟与数字混合电路，用于生成、调整时钟。
- **用途：**
    - **倍频/分频：** 输入 50MHz，输出 200MHz。
    - **相移：** 将时钟延迟 90度（常用于 SDRAM 控制器）。
    - **去抖动：** 净化输入的脏时钟。

# 二、 高速接口与通信类 (连接世界的桥梁)

随着接口速度越来越快，逻辑门（LUT）已经跟不上了，必须由硬核来实现物理层（PHY）。

## 1. 吉比特收发器 (Gigabit Transceivers - GT)

这是 FPGA 中最贵、最复杂的部分。
- **名称：** Xilinx 称为 GTP (低端), GTX, GTH, GTY (高端)。
- **能力：** 负责 **SerDes (串并转换)**。能处理 1Gbps 到 58Gbps 甚至更高的串行信号。
- **用途：** **PCIe, SATA, SFP+ (光口), USB 3.0, HDMI, DisplayPort** 的底层物理信号全靠它。
- **硬核优势：** 这种模拟混合信号电路根本无法用逻辑门实现。
## 2. PCIe 硬核 (PCI Express Integrated Block)

- **痛点：** PCIe 协议栈非常复杂（物理层、数据链路层、事务层）。如果用 Verilog 写软核，会消耗掉芯片一半的 LUT 资源，而且时序很难收敛。
- **解法：** 厂商直接把 PCIe 的协议栈做成了硬核。你只需要在 Vivado 里勾选配置一下，用户逻辑直接通过 AXI 接口和它交互即可。
## 3. 内存控制器 (Memory Controller / DDR PHY)

- **用途：** 控制 DDR3 / DDR4 / LPDDR4 内存条。
- **硬核优势：** DDR 读写对时序精度的要求是皮秒级的，且控制逻辑极其复杂。硬核 PHY 保证了信号完整性和时序校准。

# 三、 处理器与系统类 (SoC 的心脏)

在 Zynq 或 Intel SoC FPGA 系列中，这一点尤为重要。

## 1. 处理器系统 (Processing System - PS)

- **代表：** **ARM Cortex-A9** (Zynq-7000), **Cortex-A53/R5** (Zynq MPSoC)。
- **特点：** 这是一个完整的 CPU 系统，包含 Cache, MMU, 外设 (UART, SPI, I2C, CAN)。
- **注意：** 这不是 MicroBlaze（MicroBlaze 是用 LUT 搭出来的**软核**），这是真正的物理 CPU 核心。哪怕 FPGA 逻辑部分（PL）完全掉电，这个 CPU 也能独立运行 Linux。
## 2. 片上网络 (NoC - Network on Chip)

- **出现背景：** 在 Versal (ACAP) 等新一代器件中，为了解决模块间数据传输的瓶颈，芯片内部铺设了硬核的高速总线网络，不再占用普通的布线资源。

# 四、 硬核 IP vs. 软核 IP (面试必考)

面试官经常会问：“为什么我们要用硬核 IP 而不是自己写代码实现？”

| **特性**      | **硬核 IP (Hard IP)**        | **软核 IP (Soft IP)**             |
| ----------- | -------------------------- | ------------------------------- |
| **本质**      | 固化的 ASIC 电路 (硅片上的物理模块)     | Verilog/VHDL 代码 (综合后变成 LUT/FF)  |
| **性能 (速度)** | **极高** (专门优化过)             | 中等 (受限于通用布线和 LUT 性能)            |
| **面积/成本**   | **极小** (密度高)               | 较大 (LUT 利用率低)                   |
| **功耗**      | **低**                      | 较高                              |
| **灵活性**     | **低** (功能固定，参数只能微调)        | **高** (代码随便改)                   |
| **例子**      | ARM 核, SerDes, DSP48, BRAM | MicroBlaze, 自写的 I2C 控制器, 自写的加法器 |

# 五、 如何调用硬核IP？

## 方式一：代码推断 (Inference) —— “最省事，编译器自动帮你做”

这是最推荐的初学者方式，也是最通用的方式。你不需要去查阅几百页的硬核数据手册，只需要按照**标准的 Verilog 语法**写代码，综合工具（Synthesis Tool）足够聪明，会自动识别你的意图，并把逻辑映射到硬核上。

**适用场景：** 基础存储器 (BRAM)、基础运算 (DSP Slice)、简单的寄存器。

### 1. 自动推断 BRAM

你只需要定义一个大数组，不用调 IP 核：
```txt
// 这是一个标准的 Verilog 数组
reg [7:0] my_ram [0:2047]; 

always @(posedge clk) begin
    if (we)
        my_ram[addr] <= data_in; // 写
    data_out <= my_ram[addr];    // 读
end
```

**结果：** Vivado 综合时会发现这个数组太大（2048 x 8 bit），用寄存器做不划算，于是自动把它映射到 **Block RAM** 硬核中。

- 高手技巧： 如果你想强制工具使用 BRAM 或者使用 LUT (Distributed RAM)，可以在定义前加一句约束：
    ( ram_style = "block" ) reg [7:0] my_ram [0:2047];
### 2.自动推断DSP Slice

```txt
always @(posedge clk) begin
    y <= a * b + c; // 乘加运算
end
```

**结果：** 综合工具看到 `*` 和 `+`，会自动调用 **DSP48** 硬核来实现，而不是用 LUT 搭乘法器。

---

## 方式二：IP Catalog 向导 (IP Wizard) —— “图形化配置，傻瓜式生成”

对于稍微复杂一点的硬核，或者你需要用到硬核的高级功能（比如 BRAM 的双口读写、不同位宽），靠写代码推断很难描述清楚。这时候就要用官方提供的 **IP Catalog**。

**适用场景：** FIFO、Clock Wizard (PLL/MMCM)、复杂 BRAM、DDR 控制器、PCIe。

**操作流程：**

1. 在 Vivado 左侧点击 **IP Catalog**。
2. 搜索你需要的 IP（例如 "Clocking Wizard" 或 "Block Memory Generator"）。
3. **双击打开配置界面 (GUI)：** 这是一个可视化的弹窗。
    - _时钟向导：_ 你可以输入输入频率 50MHz，输出要求 200MHz，工具会自动计算 PLL 的参数。
    - _FIFO：_ 你可以选择“独立时钟 (Asynchronous)”，设置深度 1024，开启“满/空”信号标志。
4. **生成 (Generate):** 点击 OK，工具会生成一个 `.xci` 文件和一份 `.veo` (Verilog Instantiation Template) 实例化模板。
5. **复制模板：** 把模板里的代码复制到你的 Verilog 模块里即可。
---

## 方式三：原语调用 (Primitive Instantiation) —— “硬核玩家，手动连线”

这是**最底层、最硬核**的方式。当你需要使用 IOB 里的特殊资源（如 IDELAY, ODDR, BUFG）时，或者推断和 IP 核都无法满足你的特殊时序要求时，你需要直接“手动”把硬核摆在代码里。

这就像你在搭积木，不再说“我要个房子”，而是说“把这个 2x4 的红砖放在坐标 (x,y)”。

**适用场景：** I/O 接口设计 (ODDR, IDELAY, ISERDES)、全局时钟缓冲 (BUFG, BUFGMUX)、进位链手动控制。

**操作流程：**

1. **查阅文档：** 打开 Xilinx 的《UltraScale Architecture Libraries Guide》或《7 Series Libraries Guide》。
2. **使用 Language Templates (神器)：**
    - 在 Vivado 左侧菜单栏点击 `Language Templates`。
    - 找到 `Verilog -> Device Primitive Instantiation -> Artix-7 -> IO -> IDELAYE2`。
    - 工具会直接给你一段标准代码：
3. **填空：** 你只需要把你的信号名填入括号里。

```txt
// 这是一个直接调用 IDELAYE2 原语的例子
IDELAYE2 #(
   .CINVCTRL_SEL("FALSE"),          // Enable dynamic clock inversion (FALSE, TRUE)
   .DELAY_SRC("IDATAIN"),           // Delay input (IDATAIN, DATAIN)
   .HIGH_PERFORMANCE_MODE("FALSE"), // Reduced jitter ("TRUE"), Reduced power ("FALSE")
   .IDELAY_TYPE("FIXED"),           // FIXED, VARIABLE, VAR_LOAD, VAR_LOAD_PIPE
   .IDELAY_VALUE(0),                // Input delay tap setting (0-31)
   .PIPE_SEL("FALSE"),              // Select pipelined mode, FALSE, TRUE
   .REFCLK_FREQUENCY(200.0),        // IDELAYCTRL clock input frequency in MHz
   .SIGNAL_PATTERN("DATA")          // DATA, CLOCK input signal
)
IDELAYE2_inst (
   .CNTVALUEOUT(CNTVALUEOUT), // 5-bit output: Counter value output
   .DATAOUT(data_delayed),    // 1-bit output: Delayed data output
   .C(clk),                   // 1-bit input: Clock input
   .CE(1'b0),                 // 1-bit input: Clock Enable input
   .CINVCTRL(1'b0),           // 1-bit input: Dynamic clock inversion input
   .CNTVALUEIN(5'b0),         // 5-bit input: Counter value input
   .DATAIN(1'b0),             // 1-bit input: Internal delay data input
   .IDATAIN(data_in),         // 1-bit input: Data input from the I/O
   .INC(1'b0),                // 1-bit input: Increment / Decrement tap delay
   .LD(1'b0),                 // 1-bit input: Load IDELAY_VALUE input
   .LDPIPEEN(1'b0),           // 1-bit input: Enable PIPELINE register to load data input
   .REGRST(1'b0)              // 1-bit input: Active-high reset tap-delay input
);
```

## 方式四：Block Design (IP Integrator) —— “系统级拖拽，连连看”

如果你在使用 Zynq (SoC) 或者 MicroBlaze，或者需要连接极其复杂的总线（如 AXI4），写代码连线会累死人。

**适用场景：** Zynq PS 端开发、AXI 总线互联、DMA 数据搬运、复杂系统集成。

**操作流程：**

1. 创建一个 **Block Design (.bd)**。
2. 在画布上 **右键 -> Add IP**，添加 "Zynq Processing System" 和 "AXI Interconnect"。
3. **自动连线 (Connection Automation)：** Vivado 非常智能，点击上面的26绿色横条，它会自动帮你把几十根 AXI 总线信号线全部连好，处理好时钟域和位宽转换。
4. 最后，将这个 Block Design 生成一个顶层 Verilog Wrapper，就可以当作一个普通的模块去综合了。
